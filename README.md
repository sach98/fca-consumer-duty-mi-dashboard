# FCA Consumer Duty & Fair Value MI Dashboard

A regulatory Management Information (MI) framework and business analysis specification
for monitoring product performance against the UK Financial Conduct Authority's
Consumer Duty (FG22/5) Four Outcomes.

> **Data notice.** The dataset in `data/` is **synthetic and illustrative**. It is a
> four-row product-level summary written by `data/generate_duty_data.py` to exercise
> the reporting logic. It is not a real book of business, and no conclusion here is
> evidence about any real firm.
>
> **Threshold notice.** The FCA does not publish numeric pass/fail thresholds for
> claims acceptance or complaint volumes under Consumer Duty. The RAG bands used here
> are **internal review triggers** defined in `src/consumer_duty_rules.py`. They decide
> when a product line is escalated for a Fair Value review. They do not determine
> regulatory compliance.

### Book-level position (exposure-weighted)

| Measure | Book | Basis |
|---|---|---|
| Claims acceptance rate | **89.55%** | 9,313 accepted of 10,400 submitted |
| Complaints per 1k policies | **3.01** | 361 complaints across 120,000 policies |

These are weighted on the underlying counts, not averaged across product lines. The
unweighted mean of the four acceptance rates is 87.5%, which understates the book by
**2.05 percentage points** because it treats a 15,000-policy line and a 45,000-policy
line as equal contributors.

## Recommendation

**Travel Insurance should enter a Fair Value review.** It is the only line breaching an
internal trigger, and it breaches on the dimension that matters most for Consumer Duty:
complaints run at 5.47 per 1,000 against a 3.00 trigger, which is **82% above** the
book rate of 3.01.

Scope of the decision, in units the data supports:

- **Population affected:** 15,000 active policies, 12.5% of the book.
- **Claims gap:** at the book acceptance rate of 89.55%, Travel's 1,200 submitted
  claims would yield roughly 1,075 acceptances against the 948 actually accepted, a
  gap of about **127 claims**.
- **Complaint excess:** 82 complaints against roughly 45 at the book rate, an excess
  of about **37 complaints**.

Remediation cost is deliberately not stated as a number here, because this dataset
carries no cost fields. Sizing it requires the firm's average settlement value and
cost-to-serve per complaint, which should be supplied as named inputs rather than
assumed.

- **Owner:** Product Governance Lead, Travel.
- **Decision required by:** the next Products & Services Committee.
- **Escalation trigger:** any line RED on both dimensions, or RED on the same
  dimension for two consecutive reporting periods.

## Method notes and limitations

- **Four rows is not a sample.** The dataset is pre-aggregated to product level, so
  nothing here supports inference. There are no confidence intervals because there is
  no underlying claim-level variation to interval. Treat every figure as an arithmetic
  restatement of four hand-written rows, not as an estimate.
- **Exposure floor.** Any product line below 100 submitted claims is reported as
  `INSUFFICIENT_EXPOSURE` rather than being RAG-rated, so a thin book cannot be
  escalated on the strength of a handful of claims. No line in this dataset trips it.
- **Single source of truth.** The batch report, the Streamlit console and the test
  suite all import the thresholds and aggregations from
  `src/consumer_duty_rules.py`. Earlier versions of this repo defined the RAG bands
  three separate times and they disagreed with each other.
- **Not implemented:** trend over time (the dataset is a single period), claim-level
  vulnerability flags, and product-level cost data.

## Repository layout

```
data/     synthetic dataset + its generator
src/      consumer_duty_rules.py (thresholds, aggregation) + generate_mi_report.py
tests/    boundary tests, arithmetic oracles, README drift guard
docs/     BRD.md (requirements) + DAX_MEASURES.md (Power BI measure library)

app.py    Streamlit console
```

## How to run

```bash
git clone https://github.com/sach98/fca-consumer-duty-mi-dashboard.git
cd fca-consumer-duty-mi-dashboard
pip install -r requirements.txt

python3 -m src.generate_mi_report      # regenerates outputs/ and the README table
python3 -m unittest discover -s tests  # behavioural test suite
streamlit run app.py                   # interactive console
```

Regenerating the dataset itself is optional: `python3 data/generate_duty_data.py`.
